Vocabulary

    HTTP

    A standard set of rules used to transfer data over to your browser.

  • Client

    Any Workstation that requests information from a server.

  • Server

    A computer that manages resources and services to local computers when requested.

  • Web Browser

    A Program that allows you to access HTML web pages and more.

  • Website

    A website is a landing site with multiple or a single oragnized HTML page(s).

  • Web page

    A document most commonly made with the HTML that can be accessed by any web browser.

  • JavaScript

    A OOL, Object oriented language used to perform tasks that are not to be achieved using only HTML.

  • HTML

    Hypertext Markup Language is the foundation upon which webpages are presented.

  • CSS

    Cascading style sheets allow you to hone your designer aesthetics and stylize your page..

  • URL

    Uniform Resource Locator is the unique identifier for finding specific website.

  • IDE / text editor

    A computer program used for editing text related to programming and text files.

  • Command line / Terminal

    The communication between you and the computer using line by line code.

  • BASH

    This is a program that allows you to use the command line and interpret certain functions.

  • Operating system

    An operating system is the middle man between you and many programs on your computer.

  • Algorithm

    A series of instructions that are followed step by step by the computer to solve or find something.

  • Frontend vs Backend Development

    Frontend developers build with UX and design in mind. Backend developers do all the nitty gritty computational background work.

  • Programming vs Coding

    Commonly used interchangeably, coding is like scripting/command line usage, while programming is setting the foundation for which the program is going to execute the program.

  • Git vs Github

    Git is a Version control system that helps you manage and keep track of you code history. Github is a cloud based service that stores a copies of your code.

    • Github Definitions:

  • Push

    You are pushing your local content to update your repository online

  • Pull

    Checking for any changes on the current working repository

  • Fork

    Copying another user's repository for you to work on independently

  • Clone

    Copying one's repository to work offline on PC

  • Repository

    A Location to store all one's files
